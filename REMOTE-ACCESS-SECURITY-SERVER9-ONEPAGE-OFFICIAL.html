Kebijakan Keamanan Akses Remote Server 9

Tanggal Berlaku: 09 Maret 2026 | Klasifikasi: Internal | Lingkup: Administrasi server produksi

Ringkasan Keputusan

Akses remote server menggunakan model WireGuard + SSH sebagai jalur utama. Semua jalur cloud tunnel dinonaktifkan (masked) untuk menurunkan risiko pengambilalihan akses akibat kompromi akun cloud.

Perumpamaan: WireGuard adalah jalan privat ke area internal, SSH adalah pintu utama dengan kunci admin, dan rate limit firewall adalah satpam yang membatasi percobaan masuk berulang. Pintu tamu online (cloud tunnel) saat ini ditutup.

Latar Belakang Risiko

  • Potensi pembajakan akun cloud (GitHub/Microsoft).
  • Session/token cloud dapat tertinggal di server.
  • Ketergantungan berlebih pada identitas eksternal.

Implementasi Aktif

  • wg0 aktif pada 51830/udp.
  • Peer iPad khusus: 10.19.9.11.
  • SSH aktif dengan LIMIT IN 22/tcp.
  • Dua service tunnel berstatus masked.

Tingkat Keamanan

  • Risiko takeover akun cloud ke akses server: Rendah.
  • Risiko brute-force SSH publik: Terkendali.
  • Ketersediaan akses admin: Baik.

Dampak Operasional

  • Admin tetap dapat bekerja dari iPad.
  • Tidak mengganggu layanan produksi berjalan.
  • Tidak bergantung akun GitHub tertentu untuk akses server.

SOP Ringkas

  1. Nyalakan WireGuard pada perangkat admin.
  2. SSH ke IP VPN server, bukan IP publik.
  3. Lakukan aktivitas administrasi seperlunya.
  4. Putuskan VPN setelah pekerjaan selesai.

Catatan GitHub Copilot

Penggunaan Copilot tetap diperbolehkan di sisi client dan akun GitHub dapat diganti kapan saja. Perubahan akun Copilot tidak membuka akses baru ke server selama tunnel cloud tetap dinonaktifkan.
